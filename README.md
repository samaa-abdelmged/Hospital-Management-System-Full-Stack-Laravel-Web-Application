# 📝 Project Overview

The **Hospital Management System** is a full-stack web application designed to streamline hospital operations by managing patients, doctors, services, invoices, laboratories, and appointments through multiple role-based dashboards.

The system supports different user roles, each with a dedicated dashboard and permissions, while all dashboards are connected to a **single centralized backend system**.

This architecture simulates real-world hospital software systems where multiple departments interact with a unified backend infrastructure.

---

# 👥 System Roles & Dashboards

The application includes **four main dashboards**, each designed for a specific role:

### 🛠 Admin Dashboard

The administrator manages the core hospital data:

* Manage doctors
* Manage hospital departments (sections)
* Manage services
* Manage patients
* Manage insurance companies
* Manage ambulances
* Manage financial operations
* Manage appointments
* Manage posts and comments

---

### 👨‍⚕️ Doctor Dashboard

Doctors can:

* View assigned patient invoices
* Review and diagnose patients
* Add medical diagnoses
* Request laboratory tests
* Request radiology scans
* View patient medical history

---

### 🧪 Employee Dashboard

Laboratory and radiology staff can:

* View assigned medical requests
* Upload laboratory results
* Upload radiology results
* Manage completed medical tests

---

### 🧑‍🦱 Patient Dashboard

Patients can:

* View their invoices
* View laboratory results
* View radiology reports
* View payment records
* Communicate with doctors through chat
